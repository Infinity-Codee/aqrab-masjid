# تطبيق أقرب جامع

## نبذة عن المشروع
تطبيق موبايل يساعد المستخدم في العثور على أقرب جامع من موقعه الحالي باستخدام GPS والخرائط.  
عند فتح التطبيق، يحدد موقع المستخدم ثم يقارنه مع مواقع الجوامع المخزنة محليا (JSON) أو عبر قاعدة بيانات، وبعدها يعرض:
- اسم أقرب جامع
- المسافة إليه
- موقعه على الخريطة

## هدف المشروع
تسهيل وصول المستخدم إلى أقرب جامع بسرعة وبشكل واضح، خصوصا عندما يكون في منطقة غير مألوفة.

## وظائف المشروع الأساسية
- تحديد موقع المستخدم الحالي
- عرض الجوامع القريبة
- تحديد أقرب جامع
- إظهار المسافة بين المستخدم والجامع
- عرض الجامع على الخريطة
- فتح الاتجاهات للوصول إليه

## شرح أقسام المشروع
### 1) `main.dart`
نقطة بداية التطبيق وتشغيله وربط الصفحات.

### 2) `core/`
- `constants/`: الثوابت العامة (اسم التطبيق، المفاتيح،...).
- `services/`: خدمات النظام مثل الموقع والخرائط.
- `utils/`: أدوات مساعدة مثل حساب المسافات.

### 3) `data/`
- `models/`: نموذج بيانات الجامع.
- `datasource/`: مصدر البيانات (JSON / API / Firebase).
- `repositories/`: طبقة وسيطة بين المصدر وباقي التطبيق.

### 4) `presentation/`
- `screens/`: شاشات التطبيق.
- `widgets/`: مكونات واجهة قابلة لإعادة الاستخدام.
- `controllers/`: منطق الحالة والجلب والحساب.

### 5) `assets/data/mosques.json`
ملف بيانات الجوامع (الاسم، الإحداثيات، العنوان).

## الصفحات الأساسية في التطبيق
1. **Splash Screen**  
شاشة البداية (اسم التطبيق أو الشعار).
2. **Home Screen**  
تعرض موقع المستخدم، اسم أقرب جامع، المسافة، وزر عرض الخريطة.
3. **Map Screen**  
تعرض موقع المستخدم وموقع الجامع والمسار.
4. **Mosque Details Screen**  
تعرض اسم الجامع، العنوان، المسافة، وزر التوجه إليه.

## آلية عمل التطبيق
1. يفتح المستخدم التطبيق.
2. يطلب التطبيق صلاحية الوصول إلى الموقع.
3. يتم تحديد موقع المستخدم الحالي.
4. يتم تحميل بيانات الجوامع.
5. يتم حساب المسافة بين المستخدم وكل جامع.
6. يتم اختيار الجامع الأقرب.
7. يتم عرض النتيجة على الشاشة والخريطة.

## نبذة تقنية قصيرة للمناقشة
> هذا التطبيق يعتمد على GPS لتحديد موقع المستخدم، ثم يقارن هذا الموقع مع مواقع الجوامع المخزنة في قاعدة بيانات أو ملف JSON، وبعد حساب المسافة يتم عرض أقرب جامع للمستخدم مع موقعه على الخريطة.
